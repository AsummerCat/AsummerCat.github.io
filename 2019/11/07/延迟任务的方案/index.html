方案一 (数据库实现)

基于数据库 添加后 轮训查询这个 轮询

操作: 定时轮询数据库 发现超时任务 执行处理结果

方案二 (jdk的DelayedQueue实现)

基于jdk 的 延迟队列 处理 设置超时 无界队列 (系统关闭 丢失队列)

方案三(redis实现)

基于redis的Zset 有序集合 while 处理 (高可用 但是无法处理超时后的)

方案四 (基于rabbitMQ 实现 )
